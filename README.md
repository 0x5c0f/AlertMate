## What is AlertMate?

AlertMate is a full-featured web console for [Prometheus Alertmanager](https://prometheus.io/docs/alerting/latest/alertmanager/). It replaces manual YAML editing with a visual interface and adds real-time remote operations — alerts viewing, silence management, and cluster status monitoring.

Originally generated by Google AI Studio as a config editor, it has evolved into a comprehensive management tool.

### Core Capabilities

| Feature | Description |
|---------|-------------|
| **Visual Config Editor** | Manage receivers (Slack, WeChat, DingTalk, Email, Webhook, PagerDuty), routing trees, inhibit rules with tag-based UI |
| **YAML Generation & Validation** | One-click download `alertmanager.yml`, real-time validation |
| **Remote Ops** | View active alerts, create/edit/expire silences, check cluster status — all via direct Alertmanager API |
| **Pull Config** | Import existing configuration from a running Alertmanager instance |
| **AI Copilot** (optional) | Natural language config generation via Gemini or OpenAI-compatible APIs |
| **Authentication** | Optional JWT-based login to protect remote operations and AI features |
| **i18n** | English / Simplified Chinese |

### Tech Stack

React 19 · Express 4 · TypeScript · Vite 6 · Tailwind CSS 4 · Lucide Icons

## Quick Start

**Prerequisites:** Node.js ≥ 18

```bash
# Clone
git clone https://github.com/0x5c0f/AlertMate.git
cd AlertMate

# Install
npm install

# Run (default: no auth, no AI)
npm run dev
```

Open **http://localhost:3000**

## Configuration

All settings via environment variables (`.env.local` for development):

### Basic

```bash
# Alertmanager target URL (default: http://localhost:9093)
ALERTMANAGER_URL="http://your-alertmanager:9093"
```

### Authentication (optional)

```bash
# Set to enable login. Protects remote operations and AI features.
ADMIN_PASSWORD="your-strong-password"
JWT_SECRET="at-least-16-random-chars"
```

### AI Copilot (optional)

```bash
# Enable AI features
AI_ENABLED="true"

# API key (required)
AI_API_KEY="your-gemini-key"

# Or use OpenAI-compatible:
# AI_PROVIDER="custom"
# AI_BASE_URL="https://api.openai.com/v1"
# AI_API_KEY="sk-..."
# AI_MODEL="gpt-4o"
```

> ⚠️ AI_ENABLED requires ADMIN_PASSWORD to be set — this prevents unauthorized API key abuse.

## Architecture

```
┌─ Browser ───────────────────────────────────────────┐
│  React SPA                                           │
│  ├─ Config Editor (Receivers / Routes / Inhibit)     │
│  ├─ Validate & Deploy (YAML export + Pull Config)    │
│  ├─ Remote Ops (Alerts / Silences / Status)          │
│  └─ AI Copilot (optional)                            │
│                       │                              │
│  Direct Alertmanager calls for alerts/silences/status│
└───────────────────────┼──────────────────────────────┘
                        │
┌─ Node.js Server ──────┼──────────────────────────────┐
│  Express 4                                            │
│  ├─ /api/config        Read config state              │
│  ├─ /api/validate      YAML validation                │
│  ├─ /api/parse-yaml    YAML → internal format         │
│  ├─ /api/auth/*        JWT login/status               │
│  └─ /api/ai/*          AI Copilot (optional)          │
│                                                       │
│  No server-side proxy to Alertmanager (SSRF-free)     │
└───────────────────────────────────────────────────────┘
```

## Security

- **No SSRF**: All remote operations go directly from browser to Alertmanager
- **Auth optional**: Configure `ADMIN_PASSWORD` to protect remote ops and AI
- **AI protection**: `AI_ENABLED=true` enforces authentication
- **JWT**: HS256, 24h expiry, timing-safe password comparison
- **Config secrets**: Auto-sanitized in API responses when auth enabled
- **Body limit**: 1MB

> ⚠️ AlertMate's login does **not** protect the Alertmanager itself. Your Alertmanager must be independently secured (reverse proxy, network policy, or `--web.config.file`).
